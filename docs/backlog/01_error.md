ТЕХНИЧЕСКИЙ БЕКЛОГ: КЕЙС ОШИБКИ «ИСЧЕЗАЮЩЕГО РОУТЕРА» (FASTAPI + DOCKER + WINDOWS VOLUME)🚨 1. Симптомы ошибки (Что видел разработчик)Все эндпоинты старых модулей (Каталог, Склад) работают идеально и отдают 200/201.Все эндпоинты нового модуля (Касса) при любых запросах через шлюз Nginx или напрямую на порт бэкенда стабильно возвращают JSON-ответ HTTP 404 {"detail": "Not Found"}.При этом ручная проверка кода Python через консоль внутри контейнера показывает, что роутер кассы компилируется, импортируется без синтаксических ошибок и роуты физически присутствуют в объекте app.routes.🔍 2. Истинная причина (Почему это произошло)Проблема заключалась в инфраструктурной коллизии точек входа (раздвоении файлов), усугубленной спецификой кэширования Docker Desktop под Windows (WSL2):Слепая точка запуска Uvicorn: В файле Dockerfile бэкенда команда запуска была жестко зашита как CMD ["uvicorn", "main:app", ...]. Это заставляло сервер искать точку входа строго в корневом каталоге контейнера: /app/main.py.Раздвоение файлов: В проекте существовало два файла конфигурации:Старый корневой /app/main.py (который запускал Docker, и в котором не было регистрации кассы).Новый /app/src/main.py (внутри папки src/, который усердно правил разработчик, в котором касса была, но который Uvicorn игнорировал).Ловушка отладки: Инлайновые команды проверки в консоли запускались разработчиком через from src.main import app, из-за чего Python читал новый файл из папки src/ и рапортовал, что роуты кассы есть в памяти. Однако живой сервер в этот момент крутил старый корень.Конфликт импортов: Дополнительно в файле роутера кассы использовалось имя фабрики сессий async_session_factory, тогда как в database.py она называлась AsyncSessionLocal. При ленивой инициализации пакетов в FastAPI это вызывало скрытый ImportError, из-за чего фреймворк молча отбрасывал сбойный роутер при старте, не роняя весь контейнер.🛠️ 3. Как проблема была решенаОбъединение точек входа: Был создан единый эталонный файл main.py строго в корневом каталоге бэкенда (services/core/main.py), который корректно импортирует все роутеры из внутреннего пакета src.Синхронизация фабрик: Переменные работы с СУБД внутри cash.py были жестко выровнены под системное имя AsyncSessionLocal.Полный сброс кэша: Контейнеры были принудительно пересобраны с флагом --no-cache, чтобы Docker Desktop под Windows гарантированно затер старые скомпилированные слои WSL2.

"Инструкция: Если в проекте FastAPI + Docker новые эндпоинты выдают JSON 404, а старые работают, первым делом проверь Dockerfile на соответствие параметра CMD ["uvicorn", "main:app"] реальному расположению обновляемого файла main.py. Убедись, что в контейнере нет раздвоения точек входа (в корне и в папке src/). Проверь тихие ошибки импорта зависимостей в __init__.py пакетов."
