# Microservice CRM Infrastructure (FARM-Stack + Nginx)

Современная, масштабируемая и отказоустойчивая архитектура CRM-системы, построенная на базе микросервисов с использованием Docker-оркестрации. 

Проект спроектирован с упором на высокую производительность (асинхронность бэкенда) и черкое разделение зон ответственности каждого сервиса.

---

## 🚀 Инструкция по развертыванию с нуля (Для нового разработчика)

Если вы только что скачали этот репозиторий, выполните следующие шаги для локального запуска проекта на своем компьютере:

### 1. Подготовка окружения
* Убедитесь, что на вашем компьютере установлен и запущен **Docker Desktop**.
* Откройте терминал (на Windows рекомендуется **Git Bash** или **PowerShell**) и перейдите в корневую папку склонированного проекта:
  ```bash
  cd path/to/your/repository
  ```

### 2. Запуск проекта
Выполните команду для сборки всех образов и запуска контейнеров в фоновом режиме:
```bash
docker compose up --build -d
```
*Примечание: Первая сборка может занять 2–3 минуты, так как Docker будет скачивать базовые образы операционных систем (Python, Node, Postgres) и устанавливать все необходимые библиотеки.*

### 3. Проверка статуса контейнеров
Убедитесь, что все компоненты успешно запустились. Введите команду:
```bash
docker compose ps
```
В выведенной таблице должно быть **5 контейнеров**, и у каждого в графе `STATUS` должно быть написано `Up` (работает):
* `crm_nginx_gateway` (Nginx)
* `crm_frontend_react` (React)
* `crm_backend_core` (FastAPI Core)
* `crm_analyzer_service` (FastAPI Analyzer)
* `crm_postgres_db` (PostgreSQL 16)

## 🔍 Как проверить работоспособность (Тест-кейсы)

После того как все контейнеры перешли в статус `Up`, откройте ваш браузер и проверьте следующие адреса по пунктам:

1. **Проверка Пользовательского Интерфейса (Frontend)**
   * Перейдите на: `http://localhost/`
   * **Что должны увидеть**: Стартовую страницу React с фиолетовым логотипом Vite и работающим счетчиком кликов.

2. **Проверка Главного Бэкенда и Базы Данных**
   * Перейдите на: `http://localhost/api/v1/healthcheck`
   * **Что должны увидеть**: JSON-ответ вида `{"status": "healthy", "database": "connected"}`. Это подтверждает, что бэкенд видит СУБД Postgres.

3. **Проверка Автодокументации API**
   * Перейдите на: `http://localhost/docs`
   * **Что должны увидеть**: Интерактивную страницу Swagger UI со списком всех доступных эндпоинтов главного бэкенда.

4. **Проверка Модуля Аналитики**
   * Перейдите на: `http://localhost/api/v1/analytics/summary`
   * **Что должны увидеть**: JSON-ответ с тестовыми метриками продаж (`total_sales`, `active_customers` и т.д.).

5. **Проверка Связи Между Микросервисами (Логгер)**
   * После того как вы открыли пункт №2 (страничку healthcheck), главный бэкенд должен был автоматически отправить скрытый запрос в микросервис логгера.
   * Чтобы проверить, дошел ли лог, введите в терминале команду:
     ```bash
     docker logs crm_logger_service
     ```
   * **Что должны увидеть**: В консоли должна быть строчка: `[CORE] [INFO] Healthcheck endpoint was hit`. Это доказывает, что микросервисы успешно общаются между собой.

## 🏗 Что было сделано сегодня

За одну рабочую сессию была спроектирована и запущена с нуля готовая инфраструктура из 6 независимых слоев:

1. **Единая точка входа (API Gateway)**: Развернут прокси-сервер **Nginx**, который объединил все сервисы под единым портом `80`. Он бесшовно маршрутизирует трафик внутри Docker-сети.
2. **Пользовательский интерфейс (Frontend)**: Создан легковесный каркас **React + TypeScript** на базе сборщика **Vite**. Контейнеризирован с учетом live-код-моддинга через Docker Volumes.
3. **Бизнес-логика (Core Бэкенд)**: Написано ядро системы на высокопроизводительном асинхронном фреймворке **FastAPI**.
4. **Централизованный Логгер (Микросервис)**: Выделен изолированный легковесный сервис для сбора системных логов. Реализовано **асинхронное межсервисное взаимодействие** с помощью клиента `httpx`.
5. **Модуль Аналитики (Микросервис)**: Выделен отдельный изолированный сервис для тяжелых математических вычислений и построения отчетов, чтобы ресурсоемкие задачи не тормозили работу основного интерфейса CRM.
6. **База данных (Хранилище)**: Развернута СУБД **PostgreSQL 16 (Alpine)** с настроенными Docker Volumes для персистентного хранения данных между перезапусками.

## 💎 Чистота и универсальность архитектуры

Созданный каркас обладает **высокой степенью архитектурной чистоты (Production-Ready)** и готов к масштабированию:

* **Локальный Hot-Reload**: Все папки с кодом примонтированы в контейнеры через `volumes`. Вы можете редактировать код в VS Code на Windows, и изменения мгновенно применятся внутри контейнеров Docker.
* **100% готовность к деплою (Cloud-Ready)**: Архитектура полностью абстрагирована от вашей локальной машины. Чтобы запустить эту CRM на удаленном сервере (Ubuntu/CentOS), достаточно перенести туда файлы и выполнить команду запуска.
* **Чистый CORS**: Благодаря Nginx, фронтенд и бэкенды находятся на одном «домене» `localhost`, что полностью решает классическую проблему блокировок браузера по CORS.
